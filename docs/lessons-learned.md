# Lessons Learned

1. AWS Lambda timeout settings must be adjusted based on workload requirements.

2. IAM permissions should follow least-privilege principles while still providing required access.

3. CloudWatch Logs are critical for troubleshooting Lambda execution failures.

4. Stale EBS snapshots can quietly increase AWS storage costs over time.

5. Boto3 makes AWS resource automation straightforward and powerful.

6. Cost optimization projects can be implemented effectively using serverless architecture.

7. Testing failure scenarios provides valuable operational experience.
